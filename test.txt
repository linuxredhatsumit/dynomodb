Supported Actions:
fetch: Retrieve an item from a DynamoDB table.
update: Update an attribute of an item in a DynamoDB table. Supports nested JSON as a Map (e.g., details={"city": "Delhi", "zip": "110001"}) and updating specific fields within a Map, including any level of nesting (e.g., globalJourney.globalFeatureAggregates.FIRST_UPI PAYMENT.N=5 or globalJourney.level1.level2.level3.level4.level5=new_value). Automatically corrects typos in nested attribute paths (e.g., corrects globalFeatureAggrgates to globalFeatureAggregates), creates nested paths if they don’t exist, and handles DynamoDB types (e.g., stores N as a Number).
delete: Delete an item from a DynamoDB table.
Approval Gate: All operations require manual approval via the DynamoDBApproval environment.
AWS Access: Uses the EKS_Setup_Role IAM role (arn:aws:iam::483584640083:role/EKS_Setup_Role) with permissions for dynamodb:GetItem, dynamodb:UpdateItem, and dynamodb:DeleteItem.
